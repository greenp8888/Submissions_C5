AI Financial Coach Agent

This project must be realistic, secure, demo‑ready, and suitable for an investor presentation or senior engineering review.

──────────────────────────────────
PROJECT OVERVIEW
──────────────────────────────────
Build a personalized, multi‑agent AI financial advisor application.

The system securely ingests user financial data (income, expenses, debts, assets, goals) from messy real‑world documents, normalizes it into clean tabular structures, applies secure Tabular RAG for reasoning, performs deterministic financial calculations, and produces clear, actionable financial advice via a live dashboard.

The system must demonstrate:
- Multi‑agent orchestration
- Secure user‑specific data access
- Tabular RAG with data injection
- Deterministic math + LLM reasoning separation
- Workflow automation
- Live dashboarding
- Guarded financial advice

──────────────────────────────────
MANDATORY TECH STACK (DO NOT CHANGE)
──────────────────────────────────

Data Ingestion & Processing:
- PyPDF2 for PDF extraction
- Pandas for tabular data normalization and validation
- LangChain for document loaders, tools, and RAG pipelines

Multi‑Agent Orchestration:
- LangGraph for agent graphs, state transitions, and execution control

Tabular RAG & Data Injection:
- Extract clean data from messy PDFs and CSVs
- Normalize into canonical Pandas DataFrames
- Schema‑aware querying (not raw embeddings)
- Inject only authorized rows into agent context

Models & AI Inference:
- Hugging Face models for:
  - LLM reasoning and explanations
  - Embeddings (optional hybrid RAG)
  - Safety classifiers
- Use Hugging Face Inference Endpoints or self‑hosted models

Workflow Automation & Integrations:
- n8n for:
  - Event‑driven ingestion pipelines
  - Background automation
  - External API syncing
  - Notifications and alerts
  - Audit logging

Calculations & External APIs:
- Python REPL tools for deterministic calculations
- yfinance for interest rates, benchmarks, and asset context

Frontend / UX:
- Streamlit for interactive dashboards and user workflows

Security & Guardrails:
- Ozero Fine‑Grained Authorization (FGA) for secure user data access
- Llama Guard for input/output moderation and financial safety

──────────────────────────────────
SECURITY & AUTHORIZATION REQUIREMENTS
──────────────────────────────────
- All financial data is user‑scoped
- Row‑level and table‑level authorization using Ozero FGA
- Agents may only access data explicitly authorized
- Authorization checks must occur:
  1. Before Tabular RAG retrieval
  2. Before calculations
  3. Before LLM explanations
- No cross‑user leakage is permitted

──────────────────────────────────
REQUIRED AGENTS
──────────────────────────────────

1. Financial Data Ingestion Agent
   - Triggered via n8n workflows
   - Accepts PDFs, CSVs, Excel files
   - Uses PyPDF2 + Pandas + LangChain loaders
   - Extracts messy financial tables
   - Normalizes data into canonical schemas
   - Tags rows with user_id and scope for FGA enforcement

2. Tabular RAG & Data Injection Agent
   - Performs schema‑aware Pandas queries
   - Enforces Ozero FGA authorization
   - Injects only relevant rows into downstream agents
   - Prevents over‑retrieval and prompt leakage
   - Avoids generic finance reasoning when user data exists

3. Debt Analyzer Agent
   - Operates only on authorized debt tables
   - Uses Python REPL for:
     - interest calculations
     - payoff timelines
     - scenario simulations
   - Compares snowball vs avalanche strategies
   - Outputs structured debt payoff plans

4. Custom Savings Strategy Agent
   - Analyzes disposable income and cash flow
   - Calculates emergency fund targets
   - Optimizes monthly savings plans
   - Balances savings vs debt obligations

5. Budget Optimization Agent
   - Categorizes expenses from tabular data
   - Detects overspending patterns
   - Produces concrete line‑item budget suggestions
   - Explains trade‑offs and lifestyle impact clearly

6. Financial Coach Orchestrator Agent
   - Implemented using LangGraph
   - Controls execution order and dependencies
   - Resolves conflicts between agent outputs
   - Aggregates all results into a unified financial action plan
   - Delegates explanation generation to Hugging Face LLMs
