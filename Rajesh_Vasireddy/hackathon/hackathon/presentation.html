The 5 Specialist Agents

1   Log Classifier
Parses raw logs, detects ALL incidents at every severity level, extracts root cause, affected services, timestamps and error patterns
Uses LLM
2   Remediation Agent
Maps detected issues to ordered, executable fix steps with kubectl / SQL / bash commands and time estimates per step
Uses LLM
3   Cookbook Synthesizer
Converts the fix plan into a formatted Markdown runbook with quick-reference summary, rollback and prevention sections
Uses LLM
4   Notification Agent
Sends a Slack Block Kit alert for MEDIUM, HIGH and CRITICAL incidents — includes severity, root cause and runbook excerpt
Integration MEDIUM+
5   JIRA Agent
Creates a JIRA ticket with ADF-formatted description, remediation steps and runbook excerpt. CRITICAL incidents only.
Integration CRITICAL only
State shared via LangGraph
Each agent reads from and writes to IncidentState. Pipeline streams node-by-node so the UI updates live after every step.

LangGraph State Machine

▶ run_incident_pipeline()
↓
🔍 classify
↓ log_report populated
🔧 remediate
↓ remediation_plan populated
📖 cookbook
↓ cookbook_md populated
🔔 notify
↓ slack_sent set (non-fatal)
🎫 jira
↓
⏹ END — finished = true

Error short-circuit

Any node can set finished = true on error — all subsequent nodes are skipped automatically

Live streaming

graph.stream() yields a chunk after each node. The API store is updated immediately — UI polls see live step progress

Shared state

IncidentState TypedDict — each node returns a partial dict that LangGraph merges. No passing objects between agents.

REST API

FastAPI with auto-generated Swagger docs at /docs

MethodEndpointDescription
POST /analyze Submit log — returns run_id immediately (202). Pipeline runs in background.
GET /status/{run_id} Poll for live progress. Updates after every agent step via stream().
GET /runs List all submitted runs with severity, incident type and status.
GET /healthz Liveness probe — returns {"status":"ok"}. 

# Submit a log file
curl -X POST http://localhost:8000/analyze \
  -H "Content-Type: application/json" \
  -d '{"filename":"app.log","content":"..."}'

# Poll for live results
curl http://localhost:8000/status/{run_id}

Setup & Running

Local


pip install -r requirements.txt
cp .env.example .env
# Set OPENROUTER_API_KEY  (required)
# Set SLACK_WEBHOOK_URL   (optional)
# Set JIRA_URL/USER/TOKEN (optional)

make dev-api       # http://localhost:8000
make dev-frontend  # http://localhost:8501

Docker Compose


cp .env.example .env
make docker-up    # build + start
make docker-logs  # tail logs
make docker-down  # stop

make test         # pytest (14 tests)
make lint         # ruff check
